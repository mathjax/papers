What is MathJax

  • MathJax is a JavaScript library for rendering Mathematics in all browsers
  • Can take LaTeX, AsciiMath, and MathML as input
  • Generates browser output, e.g. HTML/CSS, SVG
  • De facto rendering solution of (nearly) all Mathematics on the web
  • Provides accessibility extension
  • MathJax IS NOT MathML!

MathJax Version 3

  • Full reimplementation in TypeScript
  • Faster typesetting
  • Highly modular and supporting modern Javascript technology
  • Support for more specialist LaTeX packages
    • Physics, Logic, Chemistry
  • Provides advanced accessiblity features

How is the Math rendered?

  • MathJax is loaded into the page
  • Finds the Math in a web page
  • Transforms it into HTML a browser can digest
  • Replaces the source (original LaTeX/MathML/AsciiMath) with the new structure

How is the accessibility done?

  • MathJax calls the Speech Rule Engine (SRE)
  • SRE transforms syntax into a semantic representation
  • Computes speech, navigation structure, Braille etc.

Example

$$ x = \frac{-b \pm \sqrt{b^2-4ac}}{2a} $$

  • Source in LaTeX
  • Rendered with MathJax
  • Accessibility "built-in"

What is Speech Rule Engine

  • Javascript library for translating XML into Speech (markup)
  • Open source library available at github
  • Only Math speech solution in JavaScript
  • Runs in browser, command line, as node module.
  • Initially implemented in the context of ChromeVox
  • Speech solution for: ChromeVox, MathJax, EquatIO, MathLive, \ldots

SRE's Internal Format

  • SRE uses MathML as input language
  • Can also exploit MathJax annotations derived from LaTeX
  • MathML is not very expressive
  • SRE rewrite internally into semantically improved term structure
  • From this it is easier to produce Speech or Braille output (e.g., Nemeth)
<math>
  <mi>a</mi>
  <msup>
    <mi>x</mi>
    <mn>2</mn>
  </msup>
  <mo>+</mo>
  <mi>b</mi>
  <mi>x</mi>
  <mo>+</mo>
  <mi>c</mi>
  <mo>=</mo>
  <mn>0</mn>
</math>

MathJax on the Server

  • work on the server as easily as in the browser
  • works in non-standard DOMs (e.g., markdown)
  • separable pipeline for
    • finding
    • converting
    • annotating
    • rendering math
  • E.g., LaTeX in a document can be translated into SVG with speech or Braille annotation.

LaTeX to Nemeth

Multistep pipeline for generating tactile Maths in Braille from PreTeXt Input.

  • PreTeXt books are translated automatically into Braille with
    • Liblouis for the text
    • MathJax for formulas using SRE's Nemeth translation
  • Translation pipeline is server-side only
  • Currently MathJax v2.7: awkward to work with
  • MathJax v3 is about to be integrated and should make conversion faster!

Why this Pipeline?

  • Why LaTeX or PreTeXt?
    • Give quick access open textbook and research papers
  • Why not Liblouis for everything?
    • Lacks support for advanced content: LaTeX fonts, Unicode characters (math alphabets of the upper plane), complex expressions, etc.
    • Is restricted to MathML only
  • Why not simply MathML only?
    • MathJax can deal with structures that can not even be (properly) expressed in MathML
    • Future: AMS environments should help with page layout
